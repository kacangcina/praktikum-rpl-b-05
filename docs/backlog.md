# Product Backlog
## Metode Prioritisasi: MoSCoW

| Label | Definisi |
|---|---|
| **Must-have** | Wajib ada di MVP. Tanpa fitur ini sistem tidak dapat berfungsi |
| **Should-have** | Penting dan bernilai tinggi, tetapi bisa ditunda jika waktu tidak mencukupi |
| **Could-have** | Nice to have — meningkatkan pengalaman pengguna tetapi bukan prioritas utama |
| **Won't-have** | Tidak dikerjakan semester ini. Dicatat agar ekspektasi tetap realistis |

---

## Must-have 

| No | User Story | Role | 
|---|---|---|
| 1 | Sebagai tamu, saya ingin mendaftar dan masuk ke akun menggunakan email dan kata sandi, agar saya dapat mengakses fitur lengkap aplikasi | Guest | 
| 2 | Sebagai tamu, saya ingin menjelajah dan melihat daftar resep tanpa harus login, agar saya dapat menilai manfaat aplikasi sebelum mendaftar | Guest | 
| 3 | Sebagai pengguna, saya ingin melihat detail resep lengkap beserta bahan dan langkah memasaknya, agar saya dapat mengikuti panduan memasak dengan benar | User | 
| 4 | Sebagai pengguna, saya ingin membuat dan mempublikasikan resep masakan saya sendiri, agar resep buatan saya dapat dibagikan kepada pengguna lain | User | 
| 5 | Sebagai creator, saya ingin mempublikasikan resep lengkap dengan bahan dan langkah memasaknya, agar resep saya dapat dipelajari oleh pengguna lain | Creator | 
| 6 | Sebagai admin, saya ingin memoderasi resep yang dikirimkan sebelum ditampilkan ke publik, agar konten platform sesuai standar kualitas | Admin |

## Should-have 

| ID | User Story | Role | 
|---|---|---|
| 1 | Sebagai pengguna, saya ingin memberikan komentar pada sebuah resep, agar saya dapat berbagi pengalaman dan tips dengan pengguna lain | User | 
| 2 | Sebagai pengguna, saya ingin memberikan rating atau upvote/downvote pada resep, agar resep terbaik lebih mudah ditemukan oleh pengguna lain | User | 
| 3 | Sebagai pengguna, saya ingin mencari resep berdasarkan nama masakan atau bahan yang saya miliki, agar saya dapat menemukan resep yang sesuai dengan cepat | User | 
| 4 | Sebagai pengguna, saya ingin memfilter resep berdasarkan tingkat kesulitan, agar saya hanya melihat resep yang sesuai kemampuan saya | User | 
| 5 | Sebagai pengguna, saya ingin menyimpan resep ke koleksi pribadi, agar saya dapat menemukan kembali resep favorit tanpa mencari ulang | User | 
| 6 | Sebagai pengguna, saya ingin mengikuti sesi kelas memasak live dari chef, agar saya dapat belajar secara langsung dan interaktif | User | 
| 7 | Sebagai creator, saya ingin menyelenggarakan sesi live kelas memasak, agar saya dapat mengajar dan berinteraksi langsung dengan pengguna | Creator | 
| 8 | Sebagai admin, saya ingin memverifikasi akun creator yang mengajukan permohonan, agar hanya chef kredibel yang dapat menyelenggarakan kelas live | Admin |

## Could-have

|---|---|---|
| 1 | Sebagai pengguna, saya ingin mendapatkan rekomendasi resep berbasis AI berdasarkan riwayat resep yang saya lihat dan simpan, agar saya bisa menemukan resep baru yang sesuai selera tanpa harus mencari manual | User |

## Won't-have 

| ID | Fitur | Alasan |
|---|---|---|
| 1 | Pemesanan katering online | Di luar scope aplikasi; platform difokuskan untuk belajar memasak, bukan transaksi jual beli |
| 2 | Fitur pembayaran / subscription berbayar | Kompleksitas integrasi payment gateway di luar cakupan semester ini |
| 3 | Aplikasi mobile native (iOS/Android) | Semester ini fokus pada versi web; mobile dapat dikembangkan di iterasi berikutnya |
