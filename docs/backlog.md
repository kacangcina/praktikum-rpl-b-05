# Product Backlog

## Metode Prioritisasi: MoSCoW

| Label | Definisi |
|---|---|
| **Must-have** | Wajib ada di MVP. Tanpa fitur ini sistem tidak dapat berfungsi |
| **Should-have** | Penting dan bernilai tinggi, tetapi bisa ditunda jika waktu tidak mencukupi |
| **Could-have** | Nice to have — meningkatkan pengalaman pengguna tetapi bukan prioritas utama |
| **Won't-have** | Tidak dikerjakan semester ini. Dicatat agar ekspektasi tetap realistis |
