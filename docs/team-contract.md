## 4. Standar Git Workflow
* **Branching:** Seluruh fitur dikerjakan di branch `feature/[nama-fitur]`.
* **Pull Request (PR):** Wajib melakukan PR ke branch `dev`. Reviewer wajib memberikan feedback/approve dalam 1x24 jam.
* **Standar Commit Message:** Menggunakan format `[jenis]: [deskripsi]` dalam bahasa Inggris untuk menjaga kerapian log:
  * `feat:` (fitur baru), contoh: `feat: add recipe search bar`
  * `fix:` (perbaikan bug), contoh: `fix: resolve recipe image not loading`
  * `docs:` (dokumentasi), contoh: `docs: update team-contract.md`
  * `style:` (perubahan tampilan tanpa ubah logika), contoh: `style: update landing page color`
